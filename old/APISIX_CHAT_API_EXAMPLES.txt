APISIX Gateway - Chat Completions API Examples
==============================================

All requests require APISIX consumer authentication via 'apikey' header.
Base URL: http://localhost:9080

## 1. Direct OpenAI Route (OpenAI Responses API)
curl -X POST http://localhost:9080/v1/providers/openai/chat \
  -H "Content-Type: application/json" \
  -H "apikey: test-key-12345" \
  -d '{
    "model": "gpt-4.1-2025-04-14",
    "messages": [
      {
        "role": "user",
        "content": "Hello! Can you explain APISIX in one sentence?"
      }
    ],
    "max_tokens": 100
  }'

## Notes:
- All routes use APISIX key-auth plugin for consumer authentication
- Direct routes provide maximum control and provider-specific features
- LiteLLM route provides unified API across multiple providers
- LiteLLM automatically handles provider-specific API format differences
- Available consumer keys can be managed via the portal at http://localhost:9080/portal
