- OIDC authentication flows are functional (302 redirects confirm this)
- Both environments process requests correctly

### Root Cause: IPv6 Connectivity Issues
1. **Primary Issue**: Containers attempt IPv6 connections to Microsoft OIDC endpoints
2. **Host Network**: Lacks proper IPv6 routing to Microsoft Azure networks
3. **Fallback Behavior**: System successfully falls back to IPv4
4. **User Impact**: Minimal - authentication works but with initial delay

### Recommendations

1. **IPv6 Resolution**: Configure proper IPv6 disable at DNS level

1. **SSL Verification**: Enable ssl_verify in production environments
2. **Certificate Validation**: Ensure proper CA certificate trust

#### Operational (Low Priority)
1. **Log Filtering**: Filter IPv6 error messages from monitoring

### Conclusion
The reported "500 errors" were actually transient IPv6 connectivity issues. The system self-recovers and provides the expected OIDC authentication flow. No urgent action required, but IPv6 configuration can be optimized for cleaner logs and faster response times.
