# OpenAI vs Anthropic API Protocol Comparison

Generated by `./openai-vs-anthropic-protocol.sh`

## Request Differences

| Aspect | OpenAI `/v1/chat/completions` | Anthropic `/v1/messages` |
|--------|-------------------------------|--------------------------|
| **Auth header** | `Authorization: Bearer <key>` | `x-api-key: <key>` |
| **Version header** | (none) | `anthropic-version: 2023-06-01` |
| **System message** | In messages array `{"role": "system", ...}` | Top-level `"system": "..."` field |
| **Required fields** | model, messages | model, messages, max_tokens |
| **max_tokens** | Optional (has default) | **Required** |

### Request body structure

**OpenAI:**
```json
{
  "model": "gpt-4o-mini",
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."}
  ],
  "max_tokens": 100
}
```

**Anthropic:**
```json
{
  "model": "claude-3-5-haiku-20241022",
  "system": "...",
  "messages": [
    {"role": "user", "content": "..."}
  ],
  "max_tokens": 100
}
```

## Anthropic-specific SSE Events

Anthropic uses typed events:
- `message_start` - Initial message metadata
- `content_block_start` - Start of content block
- `content_block_delta` - Incremental text
- `content_block_stop` - End of content block
- `message_delta` - Final stats (stop_reason, usage)
- `message_stop` - Stream complete

OpenAI uses simpler format:
- `data: {...}` chunks with delta content
- `data: [DONE]` to signal end

## Why Claude Code Needs Native Anthropic Protocol

1. **Different auth mechanism** - x-api-key vs Bearer token
2. **Required anthropic-version header** - No equivalent in OpenAI
3. **System message location** - Top-level field vs messages array
4. **max_tokens required** - Will fail without it
5. **Response structure completely different** - Can't just change base URL
6. **SSE event types** - Different streaming protocol
7. **Anthropic-specific endpoints** - `/v1/messages/count_tokens` has no OpenAI equivalent

## Conclusion

**Claude Code CANNOT work by just changing base URL to an OpenAI-compatible endpoint.**

The protocols are fundamentally different:
- Request format differs (system message, required fields)
- Response format differs (field names, nesting)
- Streaming protocol differs (event types)
- Auth mechanism differs

This is why we need `/provider/anthropic/*` sidecar routes that pass through native Anthropic protocol.
